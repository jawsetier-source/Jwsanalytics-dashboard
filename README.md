# 📊 F&B Multi-Unit Operations Analytics Dashboard

An interactive analytics dashboard built to demonstrate end-to-end operations reporting for a multi-unit food & beverage / convenience store business — covering **sales performance, inventory management, and rider/delivery logistics**.

🔗 **Live Demo:** [View Dashboard](https://jwsanalytics-dashboard-6mpqtfybd7cs2gdsngeeq2.streamlit.app)
📁 **Raw Data:** Sample data included in this repo (`sales_data.csv`, `inventory_data.csv`, `rider_deliveries.csv`, `sku_sales.csv`)

---

## 💡 About This Project

This dashboard was built to showcase real-world operations analyst skills gained from managing multi-unit F&B operations — including daily sales monitoring, SOP/inventory compliance, and field logistics (rider deployment). It simulates the kind of reporting an Operations Analyst or Depot Manager would use to monitor performance across multiple store locations.

## 🔍 Key Features

- **Sales Analytics** — daily sales trends, store-by-store comparison, and fast vs. slow moving SKU analysis
- **Inventory Monitoring** — stock levels vs. reorder points, with automatic flagging of critical/low stock items
- **Rider & Delivery Performance** — on-time delivery rate tracking by route and rider
- **Store-level Filtering** — drill down into any individual store's performance
- **Customizable Design** — in-app theme picker (color palette, fonts, chart styles) for presentation flexibility

## 🛠️ Built With

- **[Streamlit](https://streamlit.io/)** — web app framework
- **[Pandas](https://pandas.pydata.org/)** — data processing
- **[Plotly](https://plotly.com/python/)** — interactive charts

## 📈 Data

All data in this repository is synthetically generated to simulate realistic multi-unit F&B operations (6 stores, 90 days, 12 SKUs, 5 delivery riders). No real company data is used.

## 🚀 Running Locally

```bash
pip install -r requirements.txt
streamlit run app.py
```
